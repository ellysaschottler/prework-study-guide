HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • Use semantic tags.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Don't forget to link the style sheet.
  • Remember DRY code.

Git

  • git status: checks what branch we are currently on
  • git cd directory name: to select directory
  • git checkout main: make sure you're working from the most recent main branch
  • git pull origin main: pull the main and create your branch to work on
  • git checkout -b branch-name: creates a new branch and switches to it
  • create a new file inside the current directory (cd) by entering touch then the name of the file
  • save work to github
    • cd : navigate to the correct directory
    • git add -A: add changed files in our working directory to the staging area of Git
    • git commit -m "added CSS to project": commit to the repository
    • git pull origin main: make sure current branch is up to date before opening a pull request
    • git push origin feature/add-css: push up all of our changes

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript iscase sensitive.
  • Strings (IE a name) are to be surrounded by quotation marks, numbers and booleans are not put in quotation marks
  • Arrays are zero-indexed and sequential, each item is assigned an index according to the order in which it is stored
  • Functions must be executed after defined, order matters
  • Do hello world assignment if time permits: https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/JavaScript_basics
